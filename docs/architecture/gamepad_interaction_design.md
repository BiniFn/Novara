# Kototoro 手柄与 TV 交互体验优化计划 (Gamepad Interaction Design)

本文档归纳了为提升 Kototoro 在手柄控制（TV 端、掌机端）下的交互“丝滑感”与“直觉感”，所需要落地实施的架构层面与细节层面的改进计划。设计理念大量参考了 Wiliwili 及标准主机的 10-foot UI（十尺界面）设计规范。

## 一、核心设计原则

1.  **强烈的视觉焦点 (XY Focus Navigation)**：由于手柄脱离了指针，必须提供夸张的“发光/放大”焦点反馈，否则用户极易丢失当前操作位置。
2.  **全局统一的按键定式**：
    *   `A 键`（DPAD Center）：确认 / 呼出控制菜单。
    *   `B 键`（Back）：取消 / 返回 / 隐藏局部拦截面板。
    *   `X / Y 键`：长按替代方案 / 高阶过滤操作。
    *   `肩键 (L1/R1)`：大范围 Tab 切换 / 章节跳转。
    *   `扳机键 (L2/R2)`：分页滚动。
3.  **情景模式自动切换 (Context Shift)**：
    *   常态（列表导航模式）。
    *   全屏阅读 / 视频播放（媒体控制模式）。
    *   弹出过滤菜单 / 工具栏（局部焦点锁定期）。

## 二、当前存在的交互痛点及改进方案

### 1. 列表焦点反馈不够明显
*   **现象**：方向键切换项目时，仅依靠深色背景不足以吸引视线，显得迟钝。
*   **计划落地**：
    *   增加**比例放大**：当 `android:state_focused="true"` 时，触发卡片的 `scaleX` 和 `scaleY` 动画（放大至 1.05x）和 `elevation` 加深。
    *   增加**描边**：为通用 `selector` 中加入宽度至少为 2dp 的 Primary Color 描边以突显焦点。

### 2. 详情页焦点深渊 (DetailsActivity - ScrollView & BottomSheet 割裂)
*   **现象**：Kototoro 详情页顶部是长长的 `NestedScrollView`（简介、相关推荐），底部是收缩的章节表单 (`BottomSheet`)。一直按“向下”会卡在某个位置，无法将焦点“滑”到底部的章节列表中。
*   **计划落地**：
    *   在此界面的最底端卡片或组添加自定义键盘拦截：当处于该区域最后，若再次按下 `DPAD_DOWN`，程序强行控制 `BottomSheetBehavior.setState(STATE_EXPANDED)` 展开章节列表，并手动将焦点请求给章节列表的第一个 Item，实现无缝平移。

### 3. 可选择文本框 (SelectableTextView) 的光标黑洞
*   **现象**：由于详情简介开启了字体可选 (`textIsSelectable=true`)，方向键按“下”不会跳到下一个按钮，而是在茫茫文本的字里行间移动光标。
*   **计划落地**：
    *   动态禁用或拦截：当侦测到按键来源是 Gamepad / Joystick，且焦点落在类似控件时，禁止内部的光标位移，强行执行 `focusSearch(FOCUS_DOWN)` 跳转到外部的下一个 View。

### 4. 刷新机制 (SwipeRefreshLayout) 无法被手柄触发
*   **现象**：由于手柄无法模拟触摸下滑，所有的下拉刷新组件都面临荒废。
*   **计划落地**：
    *   提供两种冗余方案：一是利用顶部边缘反弹（列表位于顶端时，再次按 `UP` 键触发 `swipeRefreshLayout.isRefreshing = true`）；二是利用全局热键（如 `Select键`）呼出包含“强制刷新”的统一底部面板。

### 5. 长按操作缺失 (Long Press Menu)
*   **现象**：多选、删除历史记录等高度依赖手指深按的操作在手柄上很难触发（长按 `DPAD_CENTER` 不灵敏或不被设备支持）。
*   **计划落地**：
    *   利用物理手柄的额外按键，如绑定 `KEYCODE_BUTTON_X` 即视为 `LongClick`。在现有的 `AdapterDelegateClickListenerAdapter` 中分发生效。

## 三、具体操控模块化设计

### 模块 A：全局导航与肩键
*   目标：跨过焦点链，实现一键切换大板块。
*   实现：在 `MainActivity` 覆盖 `dispatchKeyEvent`。捕获 `L1 (RB)` / `R1 (LB)` 后直接在 `BottomNavigationView` 中切换 `selectedItemId` (或触发 `ActionTab`)。

### 模块 B：漫画与小说阅读界面 (Reader)
在纯阅读（已进入满屏）状态下，手柄接管为媒体模式。
*   **翻页控制**：常规 `Left/Right` 即实现左右翻页，或绑定 `L2/R2` 实现滚动形式下的 `PageUp/PageDown`。
*   **HUD 呼出与控制**：
    *   按下 `A 键` 或 `菜单键` 强制唤出顶部/底部工具栏 (`ReaderActionsView`)。
    *   呼出后焦点自动落入工具栏并锁死 (Trap Focus)。
    *   操作完毕，按 `B 键` 隐藏工具栏，回归沉浸式阅读模式并释放输入焦点回阅读画布。

### 模块 C：视频播放界面 (VideoPlayer)
*   **常态盲操**：不需要呼出进度条，直接按方向 `左 / 右` = 倒退/快进 10 秒；按下 `L1 / R1` = 上一集/下一集。
*   **OSD 控制台**：
    *   `上 / 下` 或 `A 键` = 呼出 OSD 控制层。
    *   OSD 面板激活后，进行剧集切换、画质调节。
    *   在面板内通过 `B 键` 关闭；常规下按 `B 键` 直接退出播放器，返回视频列表。

### 模块 D：全局过滤/搜索 (Filters & Search)
*   绑定专门的功能键（通常是 `Y` 键或 `Select` 键）。无论当前焦点处在信息流的哪里，按 `Y 键` 立刻弹开 `Filter Drawer` 或锁定到顶部的 `SearchBar`，进入信息过滤和筛选流。

## 四、实施建议

这些计划涉及应用底层的按键派发机制。建议从以下顺序渐进落地：

1.  **第 0 步**：引入一个基础的手柄/TV模式检测工具类，以备有条件地启用下列策略（如果不希望普通键盘受影响的话）。
2.  **第 1 步 (UI层)**：仅通过 XML Selector 或全局 Adapter 配置，修改所有的背景 `Focused` 状态，加上放大的动画特效和高对比度边框，解决核心视觉盲区。
3.  **第 2 步 (核心交互)**：完成 MainActivity 的肩键导航和 `Y键` 的呼出逻辑。
4.  **第 3 步 (场景攻坚)**：重点改造 `DetailsActivity` 这座让手柄玩家绝望的“迷宫”和 `SwipeRefreshLayout` 的兼容。
5.  **第 4 步 (业务层)**：解决阅读器和播放器这种纯净消费场景下的状态机管理。
