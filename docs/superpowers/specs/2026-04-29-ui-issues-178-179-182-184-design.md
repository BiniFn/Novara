## 背景

## #178 阅读器章节选择控件与详情页体验不一致

### 问题

阅读器内的 `ChaptersPagesSheet` 章节 tab 无法长按章节进入多选操作模式，而详情页章节列表支持长按多选（下载/删除/书签/标记当前章节）。

### 设计

复用详情页的多选模式逻辑，在 `ChaptersPagesSheet` 的章节 tab 里启用相同的长按行为。

**交互流程：**

```
正常状态：
┌─────────────────────────────┐
│  [章节] [页面] [书签]        │  ← tab 栏
├─────────────────────────────┤
│  第 1 章                    │
│  第 2 章                    │  ← 长按进入多选
└─────────────────────────────┘

多选状态（长按后）：
┌─────────────────────────────┐
│  ✕  已选 2 章  ↓ 🔖 🗑      │  ← 操作栏（替换 tab 栏）
├─────────────────────────────┤
│  ☑ 第 1 章                  │
│  ☑ 第 2 章                  │
│  ☐ 第 3 章                  │
└─────────────────────────────┘
```

**操作项（与详情页一致）：**
- 下载（所选章节均未下载时显示）
- 删除（所选章节均已下载时显示，替换"下载"）
- 书签
- 标记当前章节

**改动范围：**

| 文件 | 改动 |
|------|------|
| `ChapterItemCard.kt` | `ChapterListCard` 和 `ChapterGridCard` 已有 `onLongClick` 参数，确认透传正确 |
| `ChaptersScreen.kt` | 已有 `onItemLongClick: (ChapterListItem) -> Unit` 参数（line 57），确认在 sheet 场景下被正确传入 |
| `ChaptersPagesSheet.kt` | 在章节 tab 的 `ChaptersScreenRoot` 调用处传入 `onItemLongClick` 回调，触发多选模式；多选状态下替换 tab 栏为操作栏 |

**注意：** `ChaptersPagesSheet` 与详情页共用同一个 `ChaptersPagesViewModel`，操作结果自动同步，无需额外处理。

## #179 滚动滑块增强

### 问题

- 详情页章节列表的 `verticalScrollbar` 是 4dp 灰色细条，无法拖拽
- 阅读器底部页码 slider（Material3 Slider）视觉过于简陋

### 设计

#### A. 章节列表 verticalScrollbar 改为可拖拽 thumb

**改动文件：** `VerticalScrollbar.kt`

新增参数：
- `draggable: Boolean = true`：是否启用拖拽
- `labelProvider: ((Int) -> String)? = null`：拖拽时气泡显示的文本，传入章节序号或名称首字

**视觉规格：**
- track：1dp 宽，`onSurfaceVariant.copy(alpha = 0.2f)`，始终可见
- thumb：8dp 宽，高度 = 可见比例 × 列表高度，最小 48dp，`onSurfaceVariant.copy(alpha = 0.6f)`，圆角 4dp
- 触摸区域：水平扩展到 24dp，避免难以点中
- 拖拽气泡：thumb 左侧，`MaterialTheme.colorScheme.secondaryContainer` 背景，圆角 8dp，显示 `labelProvider` 返回的文本

**拖拽逻辑：**
- `pointerInput` 检测垂直拖拽
- 根据拖拽位置计算目标 item index：`targetIndex = (y / totalHeight * totalItems).toInt()`
- 调用 `listState.scrollToItem(targetIndex)`

**调用方更新：**
- `ChaptersScreen.kt`：传入 `labelProvider = { index -> "第 ${index + 1} 章" }`
- `PagesScreen.kt`：传入 `labelProvider = { index -> "${index + 1}" }`

#### B. 阅读器底部 slider 视觉增强

**改动文件：** `layout_reader_actions.xml`

调整 Material3 Slider 属性：
- `app:thumbRadius`：从默认值增大到 10dp
- `app:trackHeight`：从默认值增大到 4dp
- `app:labelBehavior`：保持 `floating`（拖拽时显示页码气泡，已有）

不改逻辑，仅改视觉参数。

## #182 迁移功能体验缺陷

### 问题

1. `AlternativesActivity` 是普通 Activity，没有拖拽手柄，只能靠返回手势或点击空白退出
2. 短按 item 跳转详情页后无法返回继续选择（离开了迁移流程）
3. 没有操作提示，用户不知道如何触发迁移
4. `Content` 模型无 `updatedAt` 字段，无法显示更新时间

### 设计

#### 将 `AlternativesActivity` 改为 `AlternativesSheet`（继承 `BaseAdaptiveSheet`）

**改动文件：**
- 新建 `AlternativesSheet.kt`（继承 `BaseAdaptiveSheet`，迁移 `AlternativesActivity` 的逻辑）
- 修改 `AppRouter.openAlternatives()`：改为 `AlternativesSheet().show(fragmentManager)`
- 保留 `AlternativesActivity.kt` 作为入口兼容层（`AppRouter` 改为优先 show sheet，Activity 可保留供深链接场景）

**Sheet 结构：**

```
┌─────────────────────────────┐
│         ━━━━━              │  ← BottomSheetDragHandleView
│  选择替换来源               │  ← 标题
│  短按预览详情 · 长按直接替换 │  ← 提示文案（TextView，小字，onSurfaceVariant）
├─────────────────────────────┤
│  RecyclerView（现有列表）    │
└─────────────────────────────┘
```

**交互重设计：**

| 操作 | 行为 |
|------|------|
| 短按 item（非按钮区域） | 跳转 `DetailsActivity` 预览（现有行为，保持不变） |
| 长按 item | 直接触发迁移确认对话框（新增） |
| 短按 migrate 按钮 | 触发迁移确认对话框（现有行为，保持不变） |

**改动范围：**

| 文件 | 改动 |
|------|------|
| `AlternativesSheet.kt`（新建） | 继承 `BaseAdaptiveSheet`，迁移 Activity 逻辑，加拖拽手柄和提示文案 |
| `AlternativeAD.kt` | 在 item 根 view 上添加 `setOnLongClickListener`，触发迁移确认 |
| `AppRouter.kt` | `openAlternatives()` 改为 show sheet |
| `activity_alternatives.xml` | 可复用布局，或新建 `sheet_alternatives.xml` |

**不添加更新时间**（`Content` 模型无此字段，不引入额外网络请求）。

## #184 搜索过滤交互重设计

### 问题

1. 搜索框下方的 `AssistChip` 仅展示，不可点击，标签文案不准确（"来源类型7"、"类型3"）
2. 右上角 `SearchResultsOptionsSheet` 功能完整但入口不直观
3. "高级搜索"与 `SearchKind.ADVANCED` 重合，作为独立 chip 冗余

### 设计

#### 将下方 AssistChip 改为可交互的 FilterChip

**三个 chip 职责：**

| Chip | 功能 | 点击行为 |
|------|------|---------|
| 搜索类型 | 显示当前 `SearchKind` | 循环切换 SIMPLE → TITLE → AUTHOR → TAG → ADVANCED；选中 ADVANCED 时自动展开高级搜索输入框 |
| 来源类型 | 显示已选来源数量或"全部来源" | 弹出多选 `DropdownMenu`（NATIVE/MIHON/ANIYOMI/IREADER 等） |
| 内容类型 | 显示已选内容类型或"全部内容" | 弹出多选 `DropdownMenu`（漫画/小说/视频） |

**移除独立的"高级搜索 chip"**，其功能合并到搜索类型 chip 的 ADVANCED 选项。

**视觉示例：**

```
搜索框
[全部类型 ▾] [全部来源 ▾] [全部内容 ▾]

选中 ADVANCED 后：
[高级搜索 ✓] [全部来源 ▾] [全部内容 ▾]
─────────────────────────────────────
标题: ___________
标签: ___________
作者: ___________
```

**状态同步：**

```
FilterChip.onClick → SearchViewModel.setSourceTypes() ← SearchResultsOptionsSheet
                              ↓
                    SearchSummaryRow 重组（两处 UI 同步更新）
```

右上角 `SearchResultsOptionsSheet` 保留，作为完整配置面板（含 pinnedOnly、hideEmpty 开关）。两者共享同一个 `SearchViewModel` state，无需额外同步逻辑。

**改动范围：**

| 文件 | 改动 |
|------|------|
| `SearchResultsScreen.kt` | `SearchSummaryRow`：将 `AssistChip` 改为 `FilterChip`，添加 `onClick` 回调；移除独立高级搜索 chip；来源/内容类型 chip 添加 `DropdownMenu` |
| `SearchViewModel.kt` | 无需改动（setter 方法已存在） |
| `SearchResultsOptionsSheet` | 无需改动 |
